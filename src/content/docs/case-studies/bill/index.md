BILL's developer documentation was a Zendesk help site template — unmaintained, unstructured, and built for an API that was never designed for external developers. Over five years, I rebuilt it from scratch: migrated to a modern platform, wrote a 178,000-line OpenAPI spec solo, shipped dual API versions, automated publishing with AI, and drove explosive growth in API adoption.

## Impact

- Rebuilt developer.bill.com on ReadMe — modern platform, structured IA, live API sandbox
- Wrote a 178,000-line OpenAPI spec covering 300+ endpoints, solo
- Most common developer support question ("how do I log in?") disappeared within weeks of launch
- BILL engineers adopted the site for internal sandbox testing
- Infosec used it for DOS simulation in sandbox — an unplanned but telling endorsement
- API usage grew significantly with the Postman Collection and YouTube tutorials (2025–2026)
